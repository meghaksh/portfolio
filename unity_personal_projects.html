Development Process

Game Design

As part of game design, I finalized the core game mechanic for the zombie shooter that is the first-person shooter with some enemies to kill. To make it more fun, I decided to make the enemies AI-enabled.

Environment Sandbox

To build the core mechanic of the game, I used a prototype environment using Unity Standard Assets. I used inbuilt Floor, House, Pillars, Step, and Ramp prototypes and then baked it to make it static for the enemies with NavMesh Agent to walk around.

Adding Player and Enemy

I used the FPS player controller from the Unity Standard Assets Package. Also, I attached a Carbine 3D object to it with a particle system to feedback the player about shoot. I added a capsule with a Capsule Collider, NavMeshAgent, and EnemyAI script as a placeholder for the real zombie enemy.

How Enemy AI works?

The enemy will be provoked and chase the target(player) whenever the player is within the chase range of enemy. Moreover, if the player shoots the enemy, it will get provoked. After getting provoked, it will turn around the player are start chasing. The logic for this functionality is written as part of the EnemyAI script where I am setting up isProvoked flag in above scenarios. Also, I am using Quaternion and Slerp for the smooth rotation and chase towards the player. 


/* Script supporting AI functionality of the enemy. */

public class EnemyAI : MonoBehaviour
{
	[SerializeField] Transform target;
	[SerializeField] float chaseRange;
	[SerializeField] float turnSpeed = 5f;
	NavMeshAgent navMeshAgent;
	float distanceToTarget = Mathf.Infinity;
	bool isProvoked = false;
	void Start()
	{
		//initialize the navMeshAgent
		navMeshAgent = GetComponent();
	}
	void Update()
	{
		//measuring the distance between enemy and player
		distanceToTarget = Vector3.Distance(target.position, transform.position);
		if (isProvoked)
		{
			EngageWithTarget();
		}
		else if (distanceToTarget <= chaseRange)
		{
			// if within the chaseRange, provoke the enemy.
			isProvoked = true;
		}
		
	}
	//if we shoot the enemy from wherever, it will get provoked 
	public void OnDamageTaken()
	{
		isProvoked = true;
	}
	private void EngageWithTarget()
	{
		//rotate the enemy towards the player
		EnemyFaceToPlayer();
		//start chasing until player is in stopping distance range
		if (distanceToTarget >= navMeshAgent.stoppingDistance) {
			ChaseTarget();
		}
		//if enemy is at lesser distance than stopping distance, start attacking the player.
		if(distanceToTarget <= navMeshAgent.stoppingDistance) {
			AttackTarget();
		}
	}
	private void ChaseTarget()
	{
		navMeshAgent.SetDestination(target.position);
	}
	private void AttackTarget()
	{
		Debug.Log(name + " has seeked and is destroying " + target.name);
	}
	//this function will rotate the enemy towards the player.
	private void EnemyFaceToPlayer()
	{ 
		//get the direction of the player from enemy
		Vector3 direction = (target.position - transform.position).normalized;
		// rotate smoothly in the direction
		Quaternion lookRotation = Quaternion.LookRotation(new Vector3(direction.x, 0, direction.z));
		transform.rotation = Quaternion.Slerp(transform.rotation, lookRotation, Time.deltaTime * turnSpeed);	
	}
}

Raycasting and Destroying a Player

The Player has given a script which handles raycasting and destruction of enemy logic. On the press of the "Fire1" button, it shoots at the object in front of the raycaster. Simultaneously, it will have the visual impact of the shooting as a flash.

Also, enemy has a script which will keep on checking hitPoints on enemy and once it is exceeded, enemy will die. 

	
/*The script given to player which will handle the raycast logic.*/
public class Weapon : MonoBehaviour
{
    [SerializeField] Camera FPSCamera;
    [SerializeField] float range = 100f;
    [SerializeField] float damageValue = 30f;
    [SerializeField] GameObject hitEffect;
    void Update()
    {
        if (Input.GetButtonDown("Fire1"))
        {
            Shoot();
        }  
    }
    private void Shoot()
    {
        ProcessRaycast();
    }
    private void ProcessRaycast()
    {
        RaycastHit hit;
        if (Physics.Raycast(FPSCamera.transform.position, FPSCamera.transform.forward, out hit, range))
        {
			// checking which object is hit by the raycaster and creating hit impact
            Debug.Log("I hit this thing : " + hit.transform.name);
            CreateHitImpact(hit);
        }
        else
        {
            return;
        }
    }
	//creating visual impact of the shoot.
    private void CreateHitImpact(RaycastHit hit)
    {
        GameObject impact = Instantiate(hitEffect, hit.point, Quaternion.LookRotation(hit.normal));
        Destroy(impact, 0.1f);
    }
}

Adding Terrain and Trees with materials

I replaced the sandbox prototype environment with the forest terrain I created using Unity's inbuilt terrain 3D object. I made some mountains, applied ground materials, and added trees and grass to make it look like a forest environment.

Adding Levels using ProBuilder

As part of the level design and building rooms using ProBuilder, I used the house map provided in the course I was referring to. I am working on a few more levels for this game using ProBuilder. Also, I am using this great tool to create small props like bullets, shot-gun shells, power-batteries, etc.

Adding Lights

Lighting in Unity is the most intricate part to understand for me. I wanted to give a dark environment so I removed the sun source and increased the intensity of skybox lighting for the terrain environment. For the room, I created immersive material to guide the player to move around. I am still working on the lighting part of this project as I faced difficulties baking it on the WebGL platform.

Adding Start Game and End Game UI

To tell the game story and guide user about game controls before starting the game and after the game is over, I created two UI canvas which are loaded using scripts. As part of the game loop, I gave Play Again functionality when the game is over. I used TextMesh Pro on both the canvas to render custom fonts.

Deploying the Game and Feedback from Players

After building this game for the Unity WebGL platform, I decided to deploy it as the GitHub webpage but GitHub didn't allow me to upload the fonts used in TextMeshPro. Later I deployed to itch.io which is a great platform to distribute games for indie game developers. I shared this game with my colleagues and friends. Currently, I am working on the suggestions and feedback I have received from them.

Personal attachment with this project

This project is a special one because for the first time I used the Game Engine for the full-fledged game development. The game development course on Udemy and this project made me much more confident using the Unity3D tool and it opened my vision for the future use cases I want to work on.
